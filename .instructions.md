# Agent Refactoring Instructions

You are a project cleanup specialist. Your task is to help the user refactor their AI receipt processing project from a mixed local/cloud architecture to a clean cloud-only architecture.

## Context
- **Original Setup:** Local OCR (Tesseract) + Local Ollama server + cloud fallback
- **New Setup:** Cloud vision models (Ollama cloud) + cloud LLM services
- **Goal:** Remove deprecated code, folders, and dependencies

## Your Tasks (in order)

### 1. Analysis & Decision
Ask the user to confirm which pipeline option they want:

**Option A (Minimal):** Just extract receipts and categorize them
```
Images → JSON extraction → Categorization
```
- Delete: `optimizer.py`, `ocr_json_ollama.py`, `prod_cat.py`
- Delete folders: `output_text/`, `output_json_translated/`, `output_json_optimized/`, `output/`
- Keeps: Fastest, cleanest setup

**Option B (With Translation):** Include Polish → English translation
```
Images → JSON → Translate → Categorize
```
- Keep: `ocr_json_ollama.py`
- Delete: `optimizer.py`, `prod_cat.py`
- Delete folders: `output_text/`, `output_json_optimized/`, `output/`

**Option C (Full):** Include translation AND optimization
```
Images → JSON → Translate → Optimize → Categorize
```
- Keep: Both translator and optimizer
- Delete: `prod_cat.py` only
- Delete folders: `output_text/`, `output/`

### 2. File Cleanup
Based on user's choice, delete deprecated files:
- [ ] `prod_cat.py` (old local version - always delete)
- [ ] `ocr_converter.py` (local Tesseract - delete if not needed)
- [ ] `ocr_json_conv.py` (OCR pipeline - delete if not needed)
- [ ] `optimizer.py` (only if Option A)
- [ ] `ocr_json_ollama.py` (only if Option A)

### 3. Folder Cleanup
Delete these folders (after confirming no important data):
- [ ] `output_text/` (always - Tesseract output, not needed)
- [ ] `output_json_translated/` (only if Option A)
- [ ] `output_json_optimized/` (only if Option A)
- [ ] `output/` (always - empty folder)

### 4. Configuration Updates

#### Update `requirements.txt`
- [x] Keep: `numpy`, `Pillow`, `requests`
- [x] Keep: `opencv-python-headless` IF needed for image processing
- [x] ADD: `ollama` (required by both scripts!)
- [ ] DELETE: `pytesseract` (if Option A or B)
- [ ] DELETE: `pandas` (only used by ocr_converter)
- Suggested new content:
```

```

#### Update `Dockerfile`
Remove all Tesseract dependencies:
```dockerfile

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY image-json-converter.py .
COPY prod_cat_cloud.py .
[Add other files based on pipeline choice]
RUN mkdir -p receipt_images output_json processed_images
```

#### Update `docker-compose.yml`
Replace deprecated services. Example for Option B:
```yaml
version: '3.8'
services:
  extractor:
    build: .
    volumes:
      - ./receipt_images:/app/receipt_images
      - ./output_json:/app/output_json
    command: python image-json-converter.py
    environment:
      - OLLAMA_API_KEY=${OLLAMA_API_KEY}

  translator:
    build: .
    volumes:
      - ./output_json:/app/output_json

    build: .
    volumes:
      - ./output_json_translated:/app/output_json_translated

      - OLLAMA_API_KEY=${OLLAMA_API_KEY}
```

## Conversation Flow

1. **Greet user** - Acknowledge the refactoring task
2. **Show the 3 options** - Use the REFACTORING_GUIDE.md as reference
3. **Wait for user choice** - Don't proceed until confirmed
4. **Execute cleanup** - File by file, ask for confirmation
5. **Update configs** - Dockerfile, docker-compose, requirements.txt
6. **Test** - Verify builds and syntax
7. **Report** - Summary of what was cleaned up and what remains

## Key Principles

- ✅ Always ask before deleting
- ✅ Backup old files to a `_archived/` folder if user wants
- ✅ Verify no code still references deleted files
- ✅ Keep the 3 active files: `image-json-converter.py`, `prod_cat_cloud.py`, and optional translator/optimizer
- ✅ Focus on removing the local OCR pipeline (Tesseract, local Ollama references)
- ✅ Ensure cloud credentials (OLLAMA_API_KEY) are properly documented
